Grundlegendes

Mit den LANsurfer Modules ist es ab sofort ganz einfach eigene Module in das LANsurfer Intranet einzuhängen. Die Module können bei der Installation der Software dann nach belieben aktiviert/deaktiviert werden, um das Intranet optimal auf die eigenen Wünsche und Bedürfnisse anzupassen.
Um selber ein Modul zu programmieren sind folgende Schritte zu beachten:

  • an die Verzeichnisstruktur halten
  • info.htm anpassen
  • info.inc anpassen
  • setup.inc anpassen

Hier nochmal die Vorgehensweise etwas detaillierter:

  1. Die Quelltexte inklusive Grafiken und sonstigen Dingen des neuen Moduls sind unter htdocs/Modul_Name abzulegen. Die Verzeichnisstruktur innnerhalb des Ordners kann dabei beliebig angelegt werden. Im Verzeichnis includes/Modul_Name werden wie der Name schon sagt, die Includes des Moduls abgelegt um sie vor dem Zugriff von aussen zu schützen.
  2. In der info.htm sollte der Autor des Moduls sein Modul beschreiben, als da wären die Funktionsweise des Moduls in der notwendigen Ausfürhlichkeit, die einstellbaren Optionen und zum Beispiel eine Versionsuebersicht und Kontaktmöglichkeiten.
  3. Die info.inc enthält alle anzeigbaren Informationen des neuen Moduls. Das sind zum einen der Autor und dessen E-mail, zum anderen aber auch der Name des Moduls, die Versionsnummer und das Releasedatum. Wenn das Modul die Menüleiste um einen Punkt erweitern soll, muss dem Punkt 'menuitems' ein Array hinzugefügt werden, was den Namen des Modulverzeichnisses und den Modulnamen enthält. Soll auf der Adminseite ein Link fuer Admins hinzugefügt werden, so muss dem Punkt 'menuitems_orga' ein Array hinzugefügt werden, was den Namen des Modulverzeichnisses und den Modulnamen enthält.
  4. Die setup.inc beinhaltet alle Informationen die für das LANsurfer System zur Installation des Moduls wichtig sind. Hier sind z.B. erweiterte Setup-Optionen beschrieben, die der User auswählen kann, aber vor allem wird die SQL Tabellenstruktur des Moduls definiert.

Wichtig ist, das jeglicher Ausgabetext in englischer Sprache zu formulieren ist und folgendermassen formatiert wird <? echo _("text"); ?>. Das ganze hat den Sinn, das man dann später on-the-fly die gewünschte Sprache wählen kann. Den deutschen Teil übersetzen wir dann, compilen die Sprachdatei und packen alles in ein neues Intranet Release.

Einen Blick Wert sind auf jeden Fall auch die Dateien ls_const.inc, ls_printing.inc und sql_util.inc im Includes Verzeichnis, da in diesen Dateien Konstanten und Funktionen definiert werden, die das Programmieren für LANsurfer doch erheblich vereinfachen (Datenbankzugriffe, Forms, Errorhandling, Navigationsbuttons etc.) und eine einheitliche CI und gleichmaessige Benutzerfuehrung ermöglichen. Wir bitten darum die neuen Module auch nach diesen Gesichtspunkten zu gestalten, sie also nach Aussehen und Navigationsschema den bisherigen Modulen nachzuahmen.

Verzeichnisstruktur

htdocs
Modul_Name // Quelltexte

htdocs
orga
Modul_Name // Quelltexte (Administratives Interface)

includes
Modul_Name // Includes

modules
Modul_Name
info.htm // Informationsdatei mit Modulbeschreibung (HTML ohne Header!)
info.inc // Informationen und Menueeintrag
setup.inc // Setup Optionen und SQL Table Eintraege

info.inc

 <?
  /*
  * LANsurfer - Catering Module Definition
  */
  
  $info['vendor'] = "Autor des Moduls";
  $info['vendorURL'] = "http://URL des Autors";
  $info['displayname'] = _("Name des Moduls");
  $info['version'] = "Version zb (1.0)";
  $info['builddate'] = "Releasedatum (JJJJ-MM-TT)";
  
  $info['menuitems'] = array('Modul_Verzeichnis/' => _("Modul_Name")); // Menuepunkt in der Navigation

  $info['orga_menuitems'] = array('Modul_Verzeichnis/'=> _("Modul_Name")); // Menuepunkt auf der Adminseite
?>

setup.inc

<?
  /*
  * LANsurfer - Catering Module Setup
  */
  
  $info['setup'] = array( // Setup Optionen (s. Setup Optionen Parameter)
  'Globaler Variablennname (zb LS_USE_EMOTICONS)' => array('caption' => 
  _("Optionsbeschreibung (kurz & englisch)"), 'default' => false, 
  'type' => 'bool')
  );
 
  $info['sql'] = array( // SQL Tables
  array(
  'name' => "Tabellenname",
  'fields' => "
  SQL Tabellenfelder Definitionen zb:
  id int(11) DEFAULT '0' NOT NULL auto_increment,
  date datetime DEFAULT '0000-00-00 00:00:00' NOT NULL,
  text text,
  PRIMARY KEY (id),
  KEY date_index (date)
  "
  ),
  );
?>
